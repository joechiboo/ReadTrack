# ReadTrack - 產品規格

## 1. 概述

### 願景
打造一個簡單、直覺的個人閱讀追蹤系統，幫助使用者記錄閱讀歷程、整理讀書心得、培養閱讀習慣。

### 目標
- 提供快速記錄閱讀進度的工具
- 建立個人化的讀書心得資料庫
- 視覺化呈現閱讀統計和成長軌跡
- 支援離線使用，資料完全掌握在使用者手中
- 降低記錄門檻，讓閱讀追蹤成為習慣而非負擔

### 非目標
- 不做社交功能（不是書評平台）
- 不做電子書閱讀器
- 不收集使用者資料用於商業目的
- 不需要帳號註冊或雲端同步（初期）

## 2. 使用者故事

### 主要使用者角色
**職場工作者 Joe**（38歲，IT主管）
- 想培養閱讀習慣
- 需要記錄商業、投資、技術類書籍
- 睡前有 15-30 分鐘閱讀時間
- 希望能快速記錄心得，不要太複雜

### 核心場景

#### 場景 1：記錄新書
身為讀者，我要能夠：
- 快速新增一本正在讀的書
- 記錄書名、作者、分類
- 可選擇上傳封面圖片
- 設定閱讀目標（頁數、天數）

#### 場景 2：每日閱讀記錄
身為讀者，我要能夠：
- 快速記錄今天讀了哪本書
- 記錄閱讀頁數或章節
- 快速寫下當下的想法或金句
- 記錄閱讀時間長度

#### 場景 3：撰寫讀書心得
身為讀者，我要能夠：
- 書讀完後撰寫完整心得
- 整理閱讀過程中的筆記
- 給書籍評分（1-5星）
- 標記重要金句和想法

#### 場景 4：回顧閱讀歷程
身為讀者，我要能夠：
- 查看所有讀過的書籍清單
- 篩選特定分類或時間的書籍
- 看到閱讀統計（本數、頁數、時間）
- 重溫過去的讀書心得

#### 場景 5：追蹤閱讀進度
身為讀者，我要能夠：
- 看到目前正在讀的書籍進度
- 了解距離目標還有多少
- 視覺化呈現閱讀連續天數

## 3. 功能需求

### 3.1 書籍管理
- **新增書籍**
  - 手動輸入書名、作者、出版社
  - 上傳封面圖片（可選）
  - 設定總頁數
  - 選擇分類（商業、投資、技術、小說、自我成長等）
  - 記錄購買/借閱日期

- **書籍狀態**
  - 想讀（Wishlist）
  - 正在讀（Reading）
  - 已讀完（Finished）
  - 暫停/放棄（Paused/Dropped）

- **書籍編輯與刪除**
  - 修改書籍資訊
  - 刪除書籍（含確認對話框）

### 3.2 閱讀記錄
- **快速記錄**
  - 選擇書籍
  - 記錄日期（預設今天）
  - 記錄頁數範圍（從第X頁到第Y頁）
  - 記錄閱讀時間（分鐘）
  - 快速筆記欄位（可選）

- **閱讀歷史**
  - 查看特定書籍的所有閱讀記錄
  - 時間軸呈現
  - 編輯/刪除記錄

### 3.3 讀書心得
- **心得撰寫**
  - 書籍評分（1-5星）
  - 一句話總結
  - 主要收穫（條列或段落）
  - 金句摘錄（可多筆）
  - 個人反思
  - 實際應用想法
  - 推薦給誰讀

- **心得管理**
  - 儲存草稿
  - 編輯已發布心得
  - 匯出心得（Markdown/PDF）

### 3.4 統計與視覺化
- **個人儀表板**
  - 本月閱讀統計（書籍數、頁數、時數）
  - 年度閱讀統計
  - 閱讀連續天數
  - 最常閱讀的分類

- **圖表呈現**
  - 月度閱讀趨勢圖
  - 分類分布圓餅圖
  - 年度閱讀熱力圖

- **成就系統**（可選）
  - 連續閱讀天數徽章
  - 閱讀里程碑（10本、50本、100本）

### 3.5 資料管理
- **匯出功能**
  - 匯出所有資料為 JSON
  - 匯出特定書籍心得為 Markdown
  - 匯出閱讀報告為 PDF

- **備份與還原**
  - 手動備份到本地檔案
  - 從備份檔還原

- **隱私保護**
  - 資料僅存於瀏覽器 LocalStorage/IndexedDB
  - 不上傳任何資料到伺服器

### 4.3 核心技術
- **前端**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **資料儲存**: IndexedDB (主要) + LocalStorage (設定)
- **圖表**: Chart.js 或類似輕量級圖表庫
- **構建**: Vite (快速開發)
- **測試**: Jest (單元測試)

### 4.4 效能要求
- 首次載入 < 2秒
- 新增/查詢操作 < 100ms
- 支援至少 1000 本書籍記錄
- 支援至少 10000 筆閱讀記錄

## 5. UI/UX 設計

### 5.1 主要頁面

#### 首頁 / 儀表板
- 今日閱讀快速入口
- 正在閱讀的書籍卡片（顯示進度）
- 本月統計摘要
- 閱讀連續天數

#### 書庫頁面
- 書籍清單（網格或列表視圖）
- 篩選器（狀態、分類）
- 搜尋功能
- 新增書籍按鈕

#### 閱讀記錄頁面
- 選擇書籍
- 快速記錄表單
- 歷史記錄列表

#### 心得頁面
- 心得編輯器
- 已發布心得列表
- 篩選和搜尋

#### 統計頁面
- 各種圖表
- 時間範圍選擇器
- 匯出報告功能

### 5.2 視覺設計原則
- **配色**: 溫暖、專注的色調（米色、深褐色、墨綠色）
- **字體**: 易讀的無襯線字體，重要文字用襯線
- **間距**: 充足的留白，減少視覺壓力
- **動畫**: 微妙的過渡效果（200-300ms）
- **響應式**: 手機優先設計

### 5.3 互動設計
- 快速操作優先（少於3次點擊完成記錄）
- 表單自動儲存（避免資料遺失）
- 友善的錯誤提示
- 操作後即時回饋（toast 通知）
- 支援鍵盤快捷鍵（power user）

## 6. 成功指標

### 量化指標
- **使用頻率**: 使用者每週至少使用 3 次
- **留存率**: 30 天留存率 > 60%
- **記錄完整度**: 80% 的書籍有完整心得
- **效能**: 頁面載入時間 < 2 秒

### 質化指標
- 使用者反饋「簡單易用」
- 使用者表示「幫助建立閱讀習慣」
- 使用者願意推薦給朋友

## 8. 風險與挑戰

### 技術風險
- **資料量**: IndexedDB 處理大量資料的效能
  - 緩解: 分頁載入、虛擬滾動
- **瀏覽器相容性**: 不同瀏覽器的 IndexedDB 實作差異
  - 緩解: 充分測試、提供 fallback

### 產品風險
- **使用門檻**: 記錄流程太複雜導致放棄
  - 緩解: 持續簡化 UI、提供快速入口
- **動力不足**: 缺乏持續使用的誘因
  - 緩解: 視覺化成就、閱讀連續天數

### 使用者風險
- **資料遺失**: LocalStorage/IndexedDB 被清除
  - 緩解: 定期提醒備份、自動匯出功能
